# 目录会话

## 使用前准备

先完成安装、飞书配置和后台服务启动。然后确认本机目标目录存在，并且 Claude Code 在该目录中可以正常工作。

建议绑定项目根目录，而不是某个很深的子目录。这样 Claude Code 能看到 package、配置、测试和文档等上下文。

## 推荐流程

### 绑定工作目录

在飞书里打开 `/menu`，选择新建目录会话，并输入本机目录路径。服务会把当前 chatId 和目录关联起来。

绑定后，同一个聊天里的后续消息会默认进入这个目录上下文。你不需要每次重复说明项目在哪里。

### 选择或恢复历史会话

如果该目录下已有 Claude Code 历史 session，oh-my-feishu 会展示可选会话。选择已有会话可以继续之前的上下文，创建新会话则适合开始新的任务线。

当历史 session 不存在时，系统会使用稳定 sessionId 重新建立关系，尽量保持飞书聊天和本地工作目录的连续性。

### 在飞书里继续协作

目录会话建立后，你可以直接在飞书里说“看一下这个报错”“帮我跑测试”“总结这个 PR 的风险”。Claude Code 会在绑定目录中工作，并把结果返回到飞书。

## 使用建议

高风险代码改动建议分两步：先让 Claude Code 分析和列计划，再确认是否执行修改。这样可以避免移动端误触发大范围改动。

如果一个群聊同时讨论多个项目，建议为不同项目使用不同聊天或明确切换目录，避免上下文混淆。

## 常见问题

如果 Claude Code 读不到预期文件，先确认绑定目录是不是项目根目录。也可以重新打开菜单，查看当前目录会话状态。

如果历史会话太多，优先选择最近一次和该项目相关的 session。无法判断时，创建新会话更安全。

如果服务重启后上下文丢失，检查 `~/.oh-my-feishu/workspace` 是否还存在，以及本机 Claude Code 的历史 session 是否被清理。

## 下一步

目录会话适合人工发起的项目协作。如果你希望系统自动发现错误并触发 Claude Code 分析，继续阅读 [Web Monitor 自动化](/docs/web-monitor/)。
