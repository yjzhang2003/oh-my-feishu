# 插件能力概览

## 能力模型

主服务负责消息入口和会话编排：接收飞书消息、维护聊天和本地目录的关系、渲染卡片、处理按钮回调，并在需要时调用 Claude Code。

插件 skills 负责具体飞书能力：文档怎么读写、日历怎么查、云盘怎么搜、多维表格怎么更新、消息怎么发送。它们把 lark-cli 的用法整理成 Claude Code 能理解的操作说明。

## 已覆盖的飞书域

### 沟通与协作

IM 能力用于发送消息、回复消息、查询会话和处理 reactions。它适合“把结果发到群里”“回复这条消息”“根据聊天上下文继续处理”等任务。

日历、任务、邮件和会议纪要能力适合安排会议、生成纪要、同步待办和整理沟通记录。

### 内容与知识

Docs、Wiki、Drive、Sheets、Base、Slides 和 Whiteboard 覆盖飞书里常见的知识生产场景。Claude Code 可以读取文档、创建内容、更新表格、搜索云空间资源或生成白板结构。

这些能力尤其适合“把分析结果写成文档”“更新项目排期表”“查找某个知识库页面”“生成会议复盘”等工作流。

## 推荐使用方式

用户不需要直接记住所有 lark-cli 命令。更自然的方式是在飞书里描述目标，例如：

- “帮我把这个排查结论整理成飞书文档。”
- “查一下今天下午我有没有空。”
- “把这个服务的错误摘要发到项目群。”
- “更新多维表格里这个需求的状态。”

Claude Code 会根据任务选择对应 skill。如果任务需要用户身份权限，需要先完成 `lark-cli Auth`。

## 能力边界

插件能力依赖本机 lark-cli、飞书开放平台权限和当前授权身份。没有授权的 scope、没有访问权限的文档、未发布的应用权限，都可能导致命令失败。

对高风险操作，例如批量更新、删除内容、修改生产数据，建议先让 Claude Code 输出计划或 diff，再确认执行。

## 与目录会话的关系

目录会话提供本地项目上下文，飞书 skills 提供协作平台上下文。两者组合后，Claude Code 可以一边读取代码仓库，一边把结果写回飞书文档、消息、表格或任务。

例如 Web Monitor 发现 traceback 后，Claude Code 可以在本地服务仓库里分析错误，再把根因、改动和验证结果通过飞书卡片返回。
